```java
SendEmailResponse response = client.sendEmail(
  templateId,
  emailAddress,
  personalisation,
  reference,
  emailReplyToId,
  statusCallbackUrl,
  statusCallbackBearerToken
);
```

<Panel label={<H3 appearAs={H5}>Arguments</H3>}>
<Parameters>
  <Parameter required name="templateId" kind="String">

The ID of the template

You can find this by signing into Notify and
going to the Templates page.

  </Parameter>
  <Parameter required name="emailAddress" kind="String">

The email address of the recipient you want to send to.

  </Parameter>

  <Parameter required name="personalisation" kind="Map<String, String>">

If a template has placeholder fields for personalised information such as name or application date, you must provide their values in a map. For example:

```java inline
Map<String, String> personalisation = new HashMap<>();
personalisation.put("first_name", "Amala");
personalisation.put("application_date", "2018-01-01");
```

If a template does not have any placeholder fields for personalised information, you must pass in an empty map or null.

  </Parameter>
  <Parameter required name="reference" kind="String">

A unique identifier you create. This reference identifies a single unique notification or a batch of notifications. If you do not have a reference, you must pass in an empty string or null.

  </Parameter>

  <Parameter optional name="emailReplyToId" kind="String">

This is an email reply-to address specified by you to receive replies from your users. Your service cannot go live until you set up at least one of these email addresses. To set up:

1. Sign into your Notify account.
1. Go to Settings.
1. Go to the Email section and select Manage on the Email reply-to addresses row.
1. Select Change to specify the email address to receive replies, and select Save.

```java inline nonumbers
String emailReplyToId = '8e222534-7f05-4972-86e3-17c5d9f894e2'
```

If you do not have an `emailReplyToId`, you can leave out this argument.

  </Parameter>

  <Parameter optional name="statusCallbackUrl" kind="String">

The status callback arguments allow you to receive delivery status updates to a server of your choice. If you do not use this argument and you have set up a delivery status callback URL on your Notify service, then that will be used.

This is an HTTPS URL of the server that you would like delivery status updates to be sent to.

```java inline nonumbers
String statusCallbackUrl = 'https://example.com/callback'
```

For more information on this option see the <Link to="/callbacks">callbacks</Link> documentation.

  </Parameter>

  <Parameter optional name="statusCallbackBearerToken" kind="String">

This is the bearer token that Notify will use when making a request to your server so that you can authenticate requests.

For example:

```java inline nonumbers
String statusCallbackBearerToken = 'secure-bearer-token'
```

If you provide a `statusCallbackUrl` you must provide this argument.

For more information on this option see the <Link to="/callbacks">callbacks</Link> documentation.

  </Parameter>
</Parameters>
